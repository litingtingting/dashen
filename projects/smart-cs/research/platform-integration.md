## 1️⃣ 微信公众号

### 对接方式

**类型：** 服务号（认证）

**API 文档：** https://developers.weixin.qq.com/doc/offiaccount/Message_Management/Service_Center_messages.html

### 需要准备的资料

| 项目 | 说明 | 状态 |
|------|------|------|
| 微信公众号 | 必须是**服务号**（订阅号不支持） | 待准备 |
| 微信认证 | 300 元/年企业认证费 | 待准备 |
| AppID | 公众号唯一标识 | 待获取 |
| AppSecret | 接口调用凭证 | 待获取 |
| 服务器域名 | 需备案域名 | 待准备 |
| Token | 自定义验证令牌 | 可生成 |
| EncodingAESKey | 消息加解密密钥 | 可生成 |

### 技术实现

```
1. 配置服务器地址
   - URL: https://your-domain.com/wechat/callback
   - Token: 自定义
   - EncodingAESKey: 随机生成

2. 验证服务器
   - 微信发送 GET 请求验证 signature
   - 返回 echostr 完成验证

3. 接收消息
   - 用户发消息 → 微信服务器 → 你的服务器
   - 5 秒内必须响应，否则用户看不到回复

4. 发送消息
   - 调用客服消息接口
   - 48 小时内可主动推送
```

### 限制说明

- ✅ 文本、图片、语音、视频、图文消息
- ✅ 支持多客服会话控制
- ⚠️ 用户 48 小时内未互动，不能主动推送
- ⚠️ 每月最多发 4 条模板消息

### 费用

- 微信认证：**300 元/年**
- 服务器成本：已包含
- API 调用：**免费**

## 2️⃣ 淘宝店铺

### 对接方式

**平台：** 阿里开放平台（Open Platform）

**API 文档：** https://open.taobao.com/doc.htm

### 需要准备的资料

| 项目 | 说明 | 状态 |
|------|------|------|
| 淘宝卖家账号 | 正常经营的店铺 | 待准备 |
| 阿里开放平台账号 | 注册开发者 | 待注册 |
| 应用创建 | 创建"客服工具"类应用 | 待创建 |
| App Key | 应用标识 | 待获取 |
| App Secret | 应用密钥 | 待获取 |
| 店铺授权 | 店铺授权给应用 | 待授权 |
| 备案域名 | 用于回调 | 待准备 |

### 技术实现

```
1. 注册阿里开放平台开发者
   - 网址：https://open.taobao.com
   - 选择"客服工具"应用类型

2. 创建应用
   - 填写应用信息
   - 获取 AppKey 和 AppSecret

3. 店铺授权
   - 使用 OAuth2.0 获取店铺授权
   - 获得 access_token

4. 接入千牛消息
   - 使用"阿里旺旺"相关 API
   - 监听买家消息
   - 调用发送消息接口
```

### 核心 API

| API 名称 | 功能 | 文档 |
|---------|------|------|
| `taobao.im.impr.set` | 设置旺旺分流 | [文档](https://open.taobao.com/doc.htm?source=search&docId=108322&docType=1) |
| `taobao.im.chat.get` | 获取会话记录 | [文档](https://open.taobao.com/doc.htm?source=search&docId=108321&docType=1) |
| `taobao.im.chat.send` | 发送消息 | [文档](https://open.taobao.com/doc.htm?source=search&docId=108320&docType=1) |

### 限制说明

- ⚠️ 需要店铺授权
- ⚠️ API 调用有频率限制
- ⚠️ 部分 API 需要额外申请权限
- ✅ 支持文本、图片、表情

### 费用

- 开放平台：**免费**
- API 调用：**免费**（有频率限制）
- 部分高级 API：可能收费

## 4️⃣ 小红书

### ⚠️ 重要说明

**小红书目前没有官方客服 API！**

### 可选方案

#### 方案 A：企业号后台（推荐）

- 使用小红书企业号官方后台
- 人工 + 半自动回复
- 无法完全自动化

#### 方案 B：RPA 自动化（技术难度高）

- 使用自动化工具模拟操作
- 风险：可能违反平台规则
- 不推荐用于生产环境

#### 方案 C：引导至其他平台

- 在小红书简介留微信/公众号
- 将客户引导至有 API 的平台
- 曲线实现智能客服

### 建议

**暂时不接入小红书**，优先做微信公众号和淘宝店铺，这两个有官方 API 且用户量大。

## 🏢 个体工商户影响分析

### ✅ 好消息：个体工商户**完全可以用**！

| 平台 | 个体工商户支持情况 | 说明 |
|------|------------------|------|
| **微信公众号** | ✅ 支持 | 个体户可以注册服务号，费用一样 300 元/年 |
| **淘宝店铺** | ✅ 支持 | 个体户本身就是常见卖家类型 |
| **抖音企业号** | ✅ 支持 | 个体户可以做蓝 V 认证，费用 600 元/年 |
| **阿里开放平台** | ✅ 支持 | 个体户可以注册开发者 |
| **抖音开放平台** | ✅ 支持 | 个体户可以注册开发者 |

### 📝 个体工商户 vs 有限公司

| 项目 | 个体工商户 | 有限公司 | 影响 |
|------|-----------|---------|------|
| 微信公众号认证 | ✅ 300 元/年 | ✅ 300 元/年 | 无影响 |
| 淘宝开店 | ✅ 可以 | ✅ 可以 | 无影响 |
| 抖音蓝 V | ✅ 600 元/年 | ✅ 600 元/年 | 无影响 |
| 开发者注册 | ✅ 可以 | ✅ 可以 | 无影响 |
| 备案域名 | ✅ 可以 | ✅ 可以 | 无影响 |
| 税务发票 | 简单 | 复杂 | 个体户更简单 |
| 法律责任 | 无限责任 | 有限责任 | 小项目无所谓 |

### ⚠️ 注意事项

1. **营业执照** - 确保在有效期内
2. **经营范围** - 最好包含"信息技术服务"、"软件开发"等相关内容
3. **对公账户** - 部分平台可能需要（个体户可以用法人个人账户）
4. **法人身份证** - 所有认证都需要

### 💡 建议

**个体工商户完全够用！** 不需要专门注册公司，除非：
- 业务规模很大，需要融资
- 需要开具增值税专用发票
- 想隔离个人和业务的法律责任

对于智能客服系统这种项目，个体户完全 OK～
